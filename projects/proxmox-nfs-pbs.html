Proxmox Homelab — NFS Storage & PBS Backups

End-to-end configuration for using NFS storage with a Proxmox cluster and a dedicated Proxmox Backup Server (PBS). Includes NFS export, Proxmox storage config, datastore setup, retention, and restore workflow.

1. NFS Server Setup (QNAP or Linux)

Create NFS export:

/mnt/pool1/proxmox-nfs  *(rw,sync,no_subtree_check,no_root_squash)

Restart NFS:

systemctl restart nfs-kernel-server

QNAP: Enable NFSv3 + v4 → Set “No Mapping” → Add Proxmox node IPs to Allowed Hosts.

2. Add NFS Storage to Proxmox

In Proxmox:

Datacenter → Storage → Add → NFS

Recommended settings:

ID: nfs-storage
Server: 10.10.10.50
Export: /mnt/pool1/proxmox-nfs
Content: VZDump backup, ISO, Container template
Nodes: all
Enable:
  ✓ Subdir
  ✓ Shared

Validate storage:

pvesm status

3. Deploy Proxmox Backup Server (PBS)

Install on VM or bare-metal:

apt update
apt install proxmox-backup-server

Access PBS UI:

https://10.10.10.60:8007

4. Create PBS Datastore

PBS UI:

Datastores → Add

Recommended:

Name: pbs-data
Path: /mnt/pool2/pbs
GC Schedule: daily
Prune Schedule: daily
Retention:
  Keep last: 7
  Keep daily: 14
  Keep weekly: 4
  Keep monthly: 3

For homelabs, use light pruning — PBS is dedup-heavy but can grow quickly.

5. Add PBS to Proxmox

Datacenter → Storage → Add → Proxmox Backup Server

Config:

ID: pbs
Server: 10.10.10.60
Datastore: pbs-data
User: root@pam
Password: (use API token if possible)
Fingerprint: (copy from PBS UI)
Content: VZDump backup

6. Create Backup Jobs

In Proxmox:

Datacenter → Backup → Add

Recommended homelab schedule:

Schedule: Daily @ 02:00
Mode: Snapshot
Storage: pbs
Selection:
  ✓ VMs
  ✓ LXCs
Compression: ZSTD

7. Restore Workflow

VM/LXC restore:

Proxmox → Server View → (VM) → Backup → Restore

File-level restore:

PBS → Backups → (VM/LXC) → File Restore
