Proxmox Homelab — NFS Storage & PBS Backups

End-to-end layout and configuration for a Proxmox cluster using NFS shared storage and a dedicated Proxmox Backup Server (PBS) instance. Includes storage layout, permissions, datastore design, retention, and restore workflow.

1. NFS Server Setup (QNAP or Linux)

Create an export:

/mnt/pool1/proxmox-nfs  *(rw,sync,no_subtree_check,no_root_squash)

Restart NFS service:

systemctl restart nfs-kernel-server

Note: If using QNAP → Enable NFS v3 + v4, set "No mapping", and add Proxmox node IPs to allowed hosts.

2. Add NFS Storage to Proxmox

In Proxmox UI:

Datacenter → Storage → Add → NFS

Recommended settings:

ID: nfs-storage
Server: 10.10.10.50
Export: /mnt/pool1/proxmox-nfs
Content: VZDump backup, ISO, Container template
Nodes: (all)
Enable:
    ✓ Subdir
    ✓ Shared

Validate the share by checking: 

pvesm status

3. Deploy Proxmox Backup Server (PBS)

Install PBS (recommended LXC excluded — use VM or bare-metal):

apt update
apt install proxmox-backup-server

Access web UI:

https://10.10.10.60:8007

4. Create PBS Datastore

From PBS UI:

Datastores → Add

Recommended:

Name: pbs-data
Path: /mnt/pool2/pbs
GC Schedule: daily
Prune Schedule: daily
Retention:
    Keep last: 7
    Keep daily: 14
    Keep weekly: 4
    Keep monthly: 3

Keep retention light for homelab. Adjust based on disk pool size.

5. Add PBS as Backup Target to Proxmox

Datacenter → Storage → Add → Proxmox Backup Server

Settings:

ID: pbs
Server: 10.10.10.60
Datastore: pbs-data
User: root@pam
Password: (API token recommended)
Fingerprint: (paste from PBS UI)
Content: VZDump backup

6. Create Backup Jobs

In Proxmox UI:

Datacenter → Backup → Add

Recommended homelab schedule:

Schedule: Daily @ 02:00
Mode: Snapshot
Storage: pbs
Selection: All nodes →
    ✓ VMs
    ✓ LXCs
Compression: ZSTD

7. Restore Workflow

VM/LXC Restore:

Proxmox → Server View → (VM) → Backup → Restore

File-level Restore:

PBS → Backups → (VM/LXC) → File Restore
