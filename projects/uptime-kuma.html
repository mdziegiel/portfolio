Uptime Kuma Monitoring — Homelab

Uptime Kuma provides uptime monitoring, SSL expiry alerts, Docker health checks, and status reporting. This guide shows deployment via Docker Compose inside a homelab environment.

1. Prerequisites

  • Ubuntu host with Docker + Compose
  • Optional reverse proxy (NPM, Cloudflare Tunnel, Traefik)
  • Persistent storage directory

2. Create Working Directory


mkdir -p /opt/uptime-kuma
cd /opt/uptime-kuma

3. Docker Compose Deployment

Save as docker-compose.yml:


version: "3.8"

services:
  uptime-kuma:
    image: louislam/uptime-kuma:latest
    container_name: uptime-kuma
    restart: unless-stopped
    ports:
      - "3001:3001"
    volumes:
      - ./data:/app/data
    environment:
      - TZ=America/New_York

Start the container:


docker compose up -d

Access: http://your-server-ip:3001

4. Optional — Cloudflare Zero Trust

Example Tunnel configuration:


- hostname: uptime.mrdtech.me
  service: http://10.10.10.50:3001

5. Recommended Monitor Types

  • HTTP/S — apps & dashboards
  • Ping — servers & IoT devices
  • TCP — SSH, RDP, SQL, VPN
  • DNS resolution tests
  • Docker container health
  • Push notifications (triggers)

6. Notification Integrations

Supported channels:

  • Discord
  • Telegram
  • Email / SMTP
  • Slack
  • Pushover
  • Microsoft Teams
