Uptime Kuma Monitoring — Homelab

Uptime Kuma is a lightweight, self-hosted monitoring platform used to track service uptime, SSL certificate expiry, Docker health checks, and internal/external web services. This guide covers deployment using Docker Compose on Ubuntu in a homelab environment.

1. Prerequisites

  • Ubuntu host with Docker + Docker Compose installed
  • Reverse proxy (optional): Nginx Proxy Manager, Cloudflare Tunnel, or Traefik
  • Persistent storage directory

2. Create the Uptime Kuma Directory


mkdir -p /opt/uptime-kuma
cd /opt/uptime-kuma

3. Docker Compose Deployment

Save the following as docker-compose.yml: 


version: "3.8"

services:
  uptime-kuma:
    image: louislam/uptime-kuma:latest
    container_name: uptime-kuma
    restart: unless-stopped
    ports:
      - "3001:3001"
    volumes:
      - ./data:/app/data
    environment:
      - TZ=America/New_York

Start Uptime Kuma:


docker compose up -d

Access it at: http://your-server-ip:3001

4. Optional — Expose Through Cloudflare Tunnel

If using Cloudflare Zero Trust, add an entry to your tunnel config: 


- hostname: uptime.mrdtech.me
  service: http://10.10.10.50:3001

5. Recommended Monitor Types

  • HTTP/S (Apps, Services, Dashboards)
  • Ping (Servers, IoT devices)
  • TCP (SSH, RDP, SQL, WireGuard)
  • DNS Resolution
  • Docker Container Health Checks
  • Push Notifications for automation triggers

6. Notification Integrations

Supported channels include:

  • Discord Webhooks
  • Telegram Bots
  • Email / SMTP
  • Slack
  • Pushover
  • Microsoft Teams
