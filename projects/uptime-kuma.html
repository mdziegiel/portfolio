Uptime Kuma Monitoring — Homelab

Uptime Kuma is a powerful uptime monitoring platform providing service checks, SSL expiry alerts, Docker health checks, and status notifications. This guide covers installation using Docker Compose.

1. Prerequisites

  • Ubuntu/Debian host with Docker + Docker Compose
  • Optional reverse proxy (NPM, Cloudflare Tunnel, Traefik)
  • Persistent storage directory

2. Create Working Directory

mkdir -p /opt/uptime-kuma
cd /opt/uptime-kuma

3. Deploy with Docker Compose

Create docker-compose.yml:

version: "3.8"

services:
  uptime-kuma:
    image: louislam/uptime-kuma:latest
    container_name: uptime-kuma
    restart: unless-stopped
    ports:
      - "3001:3001"
    volumes:
      - ./data:/app/data
    environment:
      - TZ=America/New_York

Start the stack:

docker compose up -d

Access at: http://your-server-ip:3001

4. Optional — Cloudflare Zero Trust

Cloudflare Tunnel config example:

- hostname: uptime.mrdtech.me
  service: http://10.10.10.50:3001

5. Recommended Monitor Types

  • HTTP/S — web apps, dashboards, health checks
  • Ping — servers, IoT devices, access points
  • TCP — SSH, RDP, SQL, VPN
  • DNS — resolution tests
  • Docker — container health monitoring
  • Push — alerts triggered by your scripts or automations

6. Notification Integrations

Supported notification services:

  • Discord
  • Telegram
  • Email / SMTP
  • Slack
  • Pushover
  • Microsoft Teams
