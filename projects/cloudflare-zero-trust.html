Cloudflare Zero-Trust for Homelab

Secure identity-aware access for Immich, RustDesk, Uptime Kuma, AdGuard Home, and other internal services — all without exposing ports to the internet. This guide shows how to deploy Cloudflare Zero-Trust and Cloudflare Tunnel for maximum homelab security.

1. Create Zero-Trust Application

Inside the Cloudflare dashboard:

  1. Go to Zero Trust → Access → Applications
  2. Click Add Application
  3. Select Self-Hosted
  4. Enter hostname (ex: immich.mrdtech.me)

This becomes the identity login page for your internal apps.

2. Configure Access Policies

Recommended rules:

  • Email login (Gmail / Microsoft)
  • Optional: Geo-blocking
  • Optional: Require Tailscale installed (device posture)

3. Install Cloudflare Tunnel

Run this on your Docker/Ubuntu host:


curl -fsSL https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation | bash

4. Configure Tunnel Services

Example config.yml:


tunnel: YOUR_TUNNEL_ID

ingress:
  - hostname: immich.mrdtech.me
    service: http://10.10.50.228:2283

  - hostname: adguard.mrdtech.me
    service: http://10.10.10.21:3000

  - hostname: rustdesk.mrdtech.me
    service: http://10.10.10.33:21117

  - service: http_status:404

5. Start & Enable Tunnel


sudo systemctl enable cloudflared
sudo systemctl start cloudflared
