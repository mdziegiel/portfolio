Cloudflare Zero-Trust for Homelab

Secure identity-aware access to internal homelab services (Immich, RustDesk, Uptime Kuma, AdGuard Home, dashboards, etc.) without exposing ports to the internet. This guide covers Cloudflare Tunnel deployment, Zero-Trust policies, and ingress routing.

1. Create Zero-Trust Application

Inside the Cloudflare Dashboard:

  1. Zero Trust → Access → Applications
  2. Select Add Application
  3. Choose Self-Hosted
  4. Enter new hostname (e.g., immich.mrdtech.me)

This becomes the Cloudflare login screen that protects your service.

2. Configure Access Policies

Recommended rules:

  • Email login (Gmail or Microsoft)
  • Optional: Geo-block by region
  • Optional: Device posture (require Tailscale installed)

3. Install Cloudflare Tunnel

Run on your Ubuntu/Docker host:

curl -fsSL https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation | bash

4. Configure Tunnel Services

Example config.yml:

tunnel: YOUR_TUNNEL_ID

ingress:
  - hostname: immich.mrdtech.me
    service: http://10.10.50.228:2283

  - hostname: adguard.mrdtech.me
    service: http://10.10.10.21:3000

  - hostname: rustdesk.mrdtech.me
    service: http://10.10.10.33:21117

  - service: http_status:404

5. Start & Enable Tunnel

sudo systemctl enable cloudflared
sudo systemctl start cloudflared
