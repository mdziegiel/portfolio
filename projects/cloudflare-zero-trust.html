Cloudflare Zero-Trust for Homelab

Identity-aware access for Immich, Wizarr, RustDesk, Uptime Kuma, AdGuard Home, and internal dashboards — all without exposing ports to the internet. This guide walks through setting up Cloudflare Zero-Trust and Cloudflare Tunnel securely in a homelab environment.

1. Create Cloudflare Zero-Trust Application

Follow these steps inside Cloudflare Dashboard:

  1. Go to Cloudflare Dashboard → Zero Trust
  2. Open Access → Applications
  3. Click Add Application
  4. Select Self-Hosted

This creates the secure identity layer Cloudflare uses for authentication.

2. Configure Access Policies

Recommended rules:

  • Email → your Gmail/Microsoft login
  • (Optional) Geo-blocking by region
  • Device posture: Require Tailscale installed (optional hardening)

3. Install Cloudflare Tunnel on Your Host

Run this command on the server hosting your services: 


curl -fsSL https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation \| bash

This installs the Cloudflare connector service (cloudflared).

4. Add Services (Immich, RustDesk, AdGuard, etc.)

Example config.yml for cloudflared:


tunnel: YOURTUNNELID
ingress:
  - hostname: immich.mrdtech.me
    service: http://10.10.50.2283

  - hostname: adguard.mrdtech.me
    service: http://10.10.10.21:3000

  - hostname: rustdesk.mrdtech.me
    service: http://10.10.10.33:21117

  - service: http_status:404

Each hostname maps to an internal service inside your homelab.

5. Start and Enable the Tunnel


sudo systemctl enable cloudflared
sudo systemctl start cloudflared
