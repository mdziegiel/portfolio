Cloudflare Zero-Trust for Homelab

Identity-aware access for Immich, Wizarr, RustDesk, Uptime Kuma, AdGuard Home, and internal dashboards — without exposing ports to the internet.

1. Create Cloudflare Zero-Trust Application

1. Go to Cloudflare Dashboard → Zero Trust 2. Open Access → Applications 3. Click Add Application 4. Choose Self-Hosted

2. Configure Policies

Recommended:

  • Email → your Gmail/Microsoft login
  • Geo-blocking (optional)
  • Device posture: Require Tailscale installed

3. Install Cloudflare Tunnel on Host

curl -fsSL https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation | bash

4. Add Services (Immich, RustDesk, AdGuard, etc.)

Example tunnel route: 

tunnel: YOURTUNNELID
ingress:
  - hostname: immich.mrdtech.me
    service: http://10.10.10.50:2283
  - hostname: adguard.mrdtech.me
    service: http://10.10.10.21
  - hostname: kuma.mrdtech.me
    service: http://10.10.10.90
  - service: http_status:404

5. Verify Access

When accessing your domain:

  • Cloudflare login should appear
  • After authentication → your app loads
