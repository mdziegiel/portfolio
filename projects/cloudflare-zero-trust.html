Cloudflare Zero-Trust for Homelab

Secure access to internal homelab services using Cloudflare Tunnel and Cloudflare Access — eliminating open ports, strengthening authentication, and improving auditability.

Overview

This configuration uses Cloudflare Zero-Trust to protect multiple internal homelab services without exposing any ports to the internet. Authentication is identity-based, MFA-enforced, and fully logged.

Instead of relying on traditional port forwarding, services such as Immich, Uptime Kuma, Gotify, AdGuard Home (UI), and Wiki/Docs are made securely reachable from anywhere using Cloudflare Tunnel.

How It Works

Cloudflare Tunnel creates an outbound-only encrypted connection from your homelab to Cloudflare’s edge.

User → Cloudflare → Cloudflare Access → Cloudflare Tunnel → Homelab Service

Deployment Steps

1. Install cloudflared

Install the Cloudflare Tunnel agent on a Linux host inside your homelab.

sudo apt update
sudo apt install cloudflared
cloudflared tunnel login
cloudflared tunnel create homelab

2. Configure Services (Applications)

In the Cloudflare Dashboard, go to: Zero Trust → Access → Tunnels

Add each homelab service as a public hostname mapped to an internal service.

Immich  
Hostname: immich.yourdomain.com  
Service: http://10.10.10.X:2283

Uptime Kuma  
Hostname: status.yourdomain.com  
Service: http://10.10.10.X:3001

Gotify  
Hostname: gotify.yourdomain.com  
Service: http://10.10.10.X:8008

AdGuard Home UI  
Hostname: adguard.yourdomain.com  
Service: http://10.10.10.21:3000

Wiki | Docs  
Hostname: docs.yourdomain.com  
Service: http://10.10.10.X:PORT

3. Configure Cloudflare Access Policies

Every service should have its own Access Policy requiring authentication.

  • Email or specific identity provider (Google, Microsoft)
  • MFA enforcement
  • Session expiration window
  • Optionally enforce device posture

4. DNS Configuration

Cloudflare automatically generates the DNS entries for your tunnel.

immich.yourdomain.com → CNAME → .cfargotunnel.com

5. Start the Tunnel

Run the tunnel service and ensure it starts on boot.

cloudflared tunnel run homelab

Benefits

  • No open ports
  • No exposure of your home IP
  • Identity-aware access (SSO)
  • MFA protection
  • Full audit logs of user actions
  • Works on any device — mobile or desktop

Troubleshooting Tips

  • Review logs: journalctl -u cloudflared
  • Ensure correct internal ports
  • Verify Access Policy email/domain
  • Check DNS propagation
