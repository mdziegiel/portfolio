Home Assistant Automations — Homelab

Home Assistant powers IoT automations, dashboards, and system monitoring across a dedicated VLAN. This guide covers Docker Compose deployment, VLAN design, and security best practices.

1. Prerequisites

  • Ubuntu host or Proxmox LXC/VM
  • Docker + Docker Compose installed
  • Dedicated VLAN for IoT devices
  • mDNS reflector enabled on firewall

2. Deploy Home Assistant with Docker Compose

Create deployment directory:


mkdir -p /opt/homeassistant
cd /opt/homeassistant

Create docker-compose.yml:


version: "3.9"

services:
  homeassistant:
    container_name: homeassistant
    image: ghcr.io/home-assistant/home-assistant:stable
    volumes:
      - ./config:/config
    environment:
      - TZ=America/New_York
    network_mode: host
    restart: unless-stopped

Note: Home Assistant must run in host mode for discovery, mDNS, and integrations. 


docker compose up -d

Access HA: http://your-server-ip:8123

3. VLAN & Network Design

Recommended segmentation:

  • IoT VLAN — cameras, sensors, smart plugs
  • Home VLAN — phones, laptops
  • HA VLAN (optional)

Allow the following between VLANs:

  • TCP 80/443
  • UDP 5353 (mDNS)
  • Vendor-specific ports (Hue, ONVIF, ESPHome)

4. Key Integrations

  • UniFi Protect — video + motion events
  • AdGuard Home — DNS filtering & telemetry
  • Uptime Kuma — service uptime monitoring
  • Tailscale — secure remote access
  • MQTT (Mosquitto) — sensors & IoT
  • ESPHome — custom automations

5. Example Automations

Automation 1 — Alert when Uptime Kuma reports a failure:


alias: Service Down Alert
trigger:
  - platform: webhook
    webhook_id: uptime_kuma_down
action:
  - service: notify.mobile_app_pixel
    data:
      message: "A monitored service is DOWN."

Automation 2 — Motion → Lights:


alias: Motion Lighting
trigger:
  - platform: state
    entity_id: binary_sensor.driveway_motion
    to: "on"
action:
  - service: light.turn_on
    entity_id: light.driveway

Automation 3 — Night mode when house unoccupied:


alias: Night Mode Away
trigger:
  - platform: state
    entity_id: group.family
    to: "not_home"
action:
  - service: climate.set_temperature
    data:
      entity_id: climate.main
      temperature: 64
  - service: lock.lock
    entity_id: lock.front_door
