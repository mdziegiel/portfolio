Home Assistant Automations — Homelab

Home Assistant powers IoT automations, dashboards, presence detection, and sensor monitoring across a dedicated IoT VLAN. This guide covers Docker deployment, network design, VLAN rules, and automation examples used in my homelab.

1. Prerequisites

  • Ubuntu host, Proxmox VM, or LXC
  • Docker + Docker Compose installed
  • Dedicated IoT VLAN (recommended)
  • mDNS reflector enabled on firewall (for discovery)

2. Deploy Home Assistant with Docker Compose

Create a working directory:

mkdir -p /opt/homeassistant
cd /opt/homeassistant

Create docker-compose.yml:

version: "3.9"

services:
  homeassistant:
    container_name: homeassistant
    image: ghcr.io/home-assistant/home-assistant:stable
    volumes:
      - ./config:/config
    environment:
      - TZ=America/New_York
    network_mode: host
    restart: unless-stopped

Note: Home Assistant requires host networking for integrations, autodiscovery, Chromecast, Apple devices, ESPHome, and mDNS. 

docker compose up -d

Access UI: http://your-server-ip:8123

3. VLAN & Network Design

Recommended segmentation:

  • IoT VLAN — Cameras, plugs, sensors, smart home devices
  • Home VLAN — PCs, phones, tablets
  • Optional HA VLAN — Dedicated host for HA + MQTT

Allow these ports between VLANs:

  • TCP 8123 (Home Assistant UI/API)
  • UDP 5353 (mDNS discovery)
  • TCP/UDP 5683 (Matter / Thread)
  • Vendor-specific ports (Hue bridge, ONVIF, ESPHome uploads)

4. Key Integrations

  • UniFi Protect — cameras + motion events
  • AdGuard Home — DNS & telemetry
  • Uptime Kuma — service uptime + alerts
  • Tailscale — remote access to HA UI
  • MQTT (Mosquitto) — sensors, switches, HA automations
  • ESPHome — embedded devices + custom sensors

5. Example Automations

Automation 1 — Service outage alert (Uptime Kuma → HA):

alias: Service Down Alert
trigger:
  - platform: webhook
    webhook_id: uptime_kuma_down
action:
  - service: notify.mobile_app_pixel
    data:
      message: "A monitored service is DOWN."

Automation 2 — Motion turns on lights:

alias: Motion Lighting
trigger:
  - platform: state
    entity_id: binary_sensor.driveway_motion
    to: "on"
action:
  - service: light.turn_on
    entity_id: light.driveway

Automation 3 — Night mode when house is empty:

alias: Night Mode Away
trigger:
  - platform: state
    entity_id: group.family
    to: "not_home"
action:
  - service: climate.set_temperature
    data:
      entity_id: climate.main
      temperature: 64
  - service: lock.lock
    entity_id: lock.front_door
