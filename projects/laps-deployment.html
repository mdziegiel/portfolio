Microsoft LAPS Deployment — Skyterra Case Study

A multi-tenant rollout of Microsoft LAPS across Skyterra-managed clients to automate local admin password rotation, strengthen security posture, and reduce privilege exposure.

1. Overview

Skyterra oversaw hundreds of Windows endpoints across dozens of tenants. Legacy local admin passwords were static, reused, and highly privileged. Microsoft LAPS was deployed to securely rotate, store, and audit local admin credentials.

2. Environment

  • Azure AD / Entra ID (cloud-only & hybrid tenants)
  • Intune-managed endpoints
  • Windows 10/11 (H1 & H2 releases)
  • Conditional Access & MFA-enforced environments

3. Deployment Strategy

The project was rolled out in three phases:

  1. Assessment — Verified prerequisites and removed conflicting legacy LAPS.
  2. Intune Policy Deployment — Configured password rotation, length, and recovery settings.
  3. Monitoring & Validation — Confirmed passwords were rotated and stored properly.

4. Required Intune Settings

Below is an example Windows LAPS policy:

Settings:
  - Enable LAPS: Enabled
  - Password Age: 7 days
  - Password Length: 16
  - Recovery: Enabled
  - Directory: Entra ID

5. Results

  • Eliminated password reuse across all tenants.
  • Reduced attack surface on local admin accounts.
  • Help Desk could securely retrieve one-time passwords via Azure portal.
  • Improved audit trails and compliance posture.
