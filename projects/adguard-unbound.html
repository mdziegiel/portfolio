AdGuard Home + Unbound Recursive DNS

A privacy-focused DNS stack using AdGuard Home for filtering and Unbound as a full recursive resolver. Supports DNSSEC, ECS-off, Split-DNS, and secure upstreams for internal services.

1. Install Unbound

Unbound acts as your internal recursive DNS resolver. Run on a separate VM/LXC (recommended). 

apt update && apt install unbound

Download the hardened config preset:

wget https://www.internic.net/domain/named.cache -O /var/lib/unbound/root.hints

2. Harden Unbound Configuration

Place this full config at:

/etc/unbound/unbound.conf.d/recursive.conf
server:
    verbosity: 1
    interface: 0.0.0.0
    port: 53
    do-ip4: yes
    do-ip6: no
    do-udp: yes
    do-tcp: yes

    # Security
    hide-identity: yes
    hide-version: yes
    harden-glue: yes
    harden-dnssec-stripped: yes
    harden-referral-path: yes
    harden-algo-downgrade: yes
    harden-large-queries: yes

    # DNSSEC
    auto-trust-anchor-file: "/var/lib/unbound/root.key"

    # Privacy
    qname-minimisation: yes
    minimal-responses: yes
    aggressive-nsec: yes
    prefetch: yes

    # Disable EDNS Client Subnet (ECS)
    send-client-subnet: "0.0.0.0/0"

    # Root servers
    root-hints: "/var/lib/unbound/root.hints"

    # Cache
    msg-cache-size: 150m
    rrset-cache-size: 300m
    cache-max-ttl: 86400
    infra-cache-numhosts: 10000

    # Performance
    num-threads: 2
    so-reuseport: yes

remote-control:
    control-enable: no

systemctl restart unbound

3. Configure AdGuard Home to Use Unbound

In AdGuard Home → Settings → DNS settings → Upstream DNS servers:

http://10.10.10.34:53

Ensure this is the ONLY upstream server. Remove Google, Cloudflare, Quad9, etc.

Recommended performance settings:

Enable DNSSEC: ✓  
Enable cache boost: ✓  
Disable IPv6 (unless fully deployed): Recommended

4. Add Split-DNS for Internal Hostnames

In AdGuard Home → DNS rewrites:

immich.mrdtech.me → 10.10.50.228
rustdesk.mydomain.com → 10.10.10.230
adguard.local → 10.10.10.21
unbound.local → 10.10.10.34

5. Verification

dig google.com @10.10.10.34
dig +dnssec cloudflare.com @10.10.10.34
dig example.com @10.10.10.21

All results should show **NO “status: SERVFAIL”** and DNSSEC should validate properly.
